## 第一阶段：需求分析与规划

### 项目背景

本项目基于《网站设计与开发流程》九阶段框架推进，目标是开发一个面向小众文化爱好者（初期聚焦效果器）的社群聚合平台，提供品牌资讯聚合与用户社区功能。

### 初始需求（四条）

1. 针对效果器、滑板、数码相机等小众品类，集成品牌官方信息
2. 按品类分板块，展示品牌新闻、折扣、固件更新等资讯
3. 按品类分区的论坛，支持用户注册、发帖、评论
4. 每条资讯下支持用户评论

### 关键决策汇总

| 决策项 | 结论 |
|--------|------|
| 初期品类 | 仅做效果器，后期扩展 |
| 目标用户 | 28–45岁男性，有消费能力，全球用户 |
| 网站语言 | 英文为主 |
| 资讯来源策略 | 爬虫 > RSS > 人工录入（三级优先级） |
| 论坛图片上传 | 暂不支持，保留扩展接口 |
| 内容审核 | 敏感词过滤 + 人工审核入口 |
| 用户激励 | 极简版：发帖数 + 注册时长，等级框架后期填充 |
| 登录方式 | Google 登录 + 邮箱注册（不支持微信/QQ） |
| 实名认证 | 不需要，保留选项 |
| 移动端 | 响应式网页，不做原生 App |
| 服务器部署 | 境外部署，无需 ICP 备案 |

## 第二阶段：潜在问题调研与决策

需求分析中识别出五个潜在风险点，本阶段对各问题进行深入调研并确定解决方案。

### 问题一：自动抓取策略（重点调研）

**法律风险评估**

- **英国：** 商业爬取处于版权法灰色地带，关键风险是违反网站 ToS（合同违约，非刑事）。最稳妥路径：只存储结构化元数据 + 链接到原文。
- **中国：** PIPL 和网安法主要针对个人信息，抓取品牌产品信息直接风险低，但大段复制原文可能涉及著作权。

**技术方案**

Playwright 抓取渲染后 HTML → LLM 提取结构化字段（名称 / 日期 / 分类 / 链接）→ 存入数据库。RSS 作为补充，Shopify 品牌默认提供，地址格式：`域名/blogs/news.atom`，合法无风险。

**robots.txt 实测结果（23个品牌）**

| 分类 | 品牌数 | 代表品牌 |
|------|--------|----------|
| 🟢 可直接爬取 | 9 | BOSS、EHX、Line 6、Neural DSP、Mooer、NUX 等 |
| 🟡 使用 RSS / 需谨慎 | 9 | JHS、Walrus、Strymon、MXR、Wampler 等 |
| 🔴 暂不处理 | 5 | Kemper、Chase Bliss、Joyo、TC Electronic、Behringer |

> ⚠️ EQD 和 Old Blood Noise 专门禁止 AI 爬虫（ClaudeBot、GPTBot 等），User-Agent 避免包含 AI 关键词。

**本问题产出文件**

| 文件 | 说明 |
|------|------|
| `check_robots_txt.py` | 批量检测品牌 robots.txt 的 Python 脚本 |
| `robots_report.txt` | 实测结果原始数据（用户本地生成） |
| `爬取策略文档.docx` | 三分类策略文档，含彩色表格与行动建议 |

### 问题二：积分体系复杂度

✅ **决策：** 初期做极简版，只显示发帖数 + 注册时间，后期按需扩展完整积分/等级/勋章系统。

### 问题三：第三方登录资质

| 平台 | 审核时长 | 费用 | 关键前提 |
|------|----------|------|----------|
| 微信登录 | 约 7–10 工作日 | 300元/年 | 需企业营业执照，个人开发者无法申请 |
| QQ 登录 | 约 5–7 工作日 | 同上 | 同上，可与微信同步申请 |
| Google 登录 | 即时（测试模式） | 免费 | 无需资质，但中国大陆用户无法使用 |

✅ **决策：** 采用 Google 登录 + 邮箱注册，规避企业资质门槛，与境外部署方向一致。

### 问题四：移动端适配工作量

✅ **决策：** 桌面 + 手机浏览器同步推进响应式设计，不做原生 App。重点注意：汉堡菜单、表格适配、图片懒加载。

### 问题五：GDPR 合规

面向全球用户，需满足欧盟数据保护基本要求。已纳入非功能需求，于 PRD v0.2 中记录，境外部署可统一处理。
