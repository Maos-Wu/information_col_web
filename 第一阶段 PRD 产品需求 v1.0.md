# 第一阶段 PRD 产品需求 v1.0
**小众文化 · 数码产品 · 社群聚合平台**
**整理日期：** 2026年2月24日

---

## 一、项目概述

打造一个面向全球小众文化与数码产品爱好者的官方信息聚合与社群交流平台。平台整合品牌官方资讯（新品发布、折扣活动、固件/驱动更新等），并提供按品类划分的社区论坛，支持用户注册、发帖与评论互动。

MVP 阶段聚焦单一品类：**吉他效果器**。待产品模式跑通后，再向滑板、相机、黑胶唱片机、机械键盘等品类扩展。

---

## 二、目标用户

- **年龄：** 28 - 45 岁
- **性别：** 以男性为主
- **特征：** 具备消费能力，愿意为小众爱好付费；对所在品类有深度兴趣
- **地区：** 面向全球，网站内容以英文为主
- **行为：** 高频关注品牌动态，活跃于小众社群，对品牌有较高忠诚度

---

## 三、核心功能需求

### 3.1 品类板块 / 信息聚合

- 网站按品类划分板块，MVP 阶段仅开放「效果器」板块
- 每个板块展示品牌官方资讯：新品发布、折扣活动、固件/驱动更新
- 每条资讯底部支持用户评论

**资讯抓取三级优先级策略：**

1. 直接爬取品牌官网（需通过 robots.txt 合规检测）
2. RSS / Atom 订阅（适用于提供订阅源的品牌，如 Shopify 平台品牌）
3. 人工录入（适用于无法自动抓取的品牌或特殊内容）

### 3.2 论坛社区

- 论坛分区与品类板块对应
- 支持用户注册与登录
- 用户可发布纯文字帖子（图片上传暂不支持，后期可扩展）
- 帖子支持评论与多层回复
- 内容审核：敏感词过滤 + 保留人工审核入口

### 3.3 用户账户系统

- 注册 / 登录 / 修改密码
- 登录方式：**Google 第三方登录 + 邮箱注册**（不支持微信/QQ）
- 实名认证：不需要，保留后期扩展选项
- 个人主页：展示发帖记录与基础信息

### 3.4 用户激励体系（极简版）

- 初期只展示：发帖数量 + 注册时长
- 等级体系框架预留，后期迭代填充规则
- 暂不实现积分兑换、勋章等复杂机制

---

## 四、非功能需求

| 维度 | 要求 |
|------|------|
| 性能 | 首屏加载 3 秒以内，初期支持百人级并发 |
| 桌面端 | 支持 Chrome、Safari、Edge |
| 移动端 | 响应式网页适配，不做原生 App |
| 安全 | 密码加密存储、验证码防垃圾注册、SQL注入/CSRF防护 |
| 合规 | 遵守 GDPR 基本要求；境外部署，无需ICP备案 |

---

## 五、当前版本范围外（待后期迭代）

- 图片 / 视频上传
- 中文界面 / 多语言支持
- 微信 / QQ 登录
- 完整积分 / 勋章激励体系
- 品牌方后台管理系统
- 商品购买 / 电商功能
- 移动端原生 App
- 效果器以外的品类板块

---

## 六、关键决策汇总

| 决策项 | 结论 |
|--------|------|
| 初期品类 | 仅做效果器，后期扩展 |
| 目标用户地区 | 面向全球，英文为主 |
| 资讯来源策略 | 爬虫 > RSS > 人工录入 |
| 论坛图片上传 | 暂不支持，保留扩展接口 |
| 内容审核 | 敏感词过滤 + 人工审核入口 |
| 用户激励 | 极简版（发帖数 + 注册时长） |
| 登录方式 | Google 登录 + 邮箱注册 |
| 实名认证 | 不需要，保留选项 |
| 移动端 | 响应式网页，不做原生 App |
| 服务器部署 | 境外，无需 ICP 备案 |
